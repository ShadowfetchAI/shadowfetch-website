# Security Policy

## Reporting a vulnerability

Please do not open public GitHub issues for security-sensitive problems.

If you discover a vulnerability affecting ShadowFetch News, send a private report with:

- a short description of the issue
- steps to reproduce it
- the affected page, route, or component
- any suggested mitigation, if you have one

## Scope

Security reports are especially helpful for issues involving:

- Cloudflare Functions
- D1-backed counter behavior
- deployment or secret handling
- script-driven content generation
- third-party feed ingestion

## Response goals

Reasonable efforts will be made to:

- confirm the report
- assess severity
- fix or mitigate valid issues
- avoid publishing sensitive details before a patch is ready
