Common questions

What customers typically ask

Where do the PDFs go?

Single capture can export through the native Files sheet. Rapid capture writes each PDF immediately into the approved Fast PDF folder.

Why does rapid capture require a folder first?

Rapid mode is built to avoid reopening the Files sheet between pages. The customer approves one location first so the app can save immediately after each shot.

Does the app use iCloud Drive?

Only if the customer picks an iCloud Drive location in Files. Any sync is then handled by Apple through Files and the customer’s Apple account.

Why does the app ask for camera access?

Fast PDF uses the device camera to capture paper pages and turn each image into a local PDF. The core workflow requires a real iPhone or iPad camera.

Does Fast PDF create an account?

No. Fast PDF does not require sign-in, does not create a Shadowfetch account, and does not upload document images to a vendor server.

Troubleshooting

First things to check

  • If the camera does not open, confirm camera permission is allowed for Fast PDF in iOS Settings.
  • If rapid capture stops saving into the folder, choose the Fast PDF folder again so iOS can refresh the folder permission.
  • If a save fails, check that the selected Files location is still available and has free space.
  • If you want maximum flexibility instead of speed, switch back to Files Each Time and export one document at a time.
