Privacy-first utility

Receipt scanning that stays in the customer’s own Files storage.

Receipt to PDF is a direct iPhone and iPad utility built for one job: scan a paper receipt with Apple’s native document camera, rename the file, and save the PDF locally. No account. No vendor cloud. No tracking.

Native capture

Uses VNDocumentCameraViewController for multi-page receipt scanning on real iPhone and iPad hardware.

Flexible save flow

Either export through the Files save sheet every time or approve a parent location once and let the app create a Receipts folder.

Local privacy

An optional 4-digit passcode can lock the app without introducing an account system or remote vault.

How it works

A short flow with no account detours.

1. Scan the receipt

The user taps Scan Receipt and the app opens Apple’s document camera for edge detection and capture.

2. Rename the PDF

The app suggests a file name, but the user can rename it before export.

3. Choose the save behavior

Use Files for every export or pick a parent location once so the app can create and reuse a Receipts folder after permission is granted.

4. Keep the file under the customer’s control

If the user chooses iCloud Drive, sync is handled by Apple’s Files stack and Apple account, not by Shadowfetch infrastructure.
