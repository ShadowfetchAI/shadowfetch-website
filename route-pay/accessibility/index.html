Accessibility

Current accessibility expectations.

  • Route Pay uses standard iOS controls and a portrait-first layout for the core workflow.
  • The main tasks do not depend on audio or video content.
  • The app is intended to work with standard Dynamic Type scaling and iOS appearance settings.
Validation note

Claim only what is fully tested.

Before selecting accessibility labels in App Store Connect, validate the exact shipping build on supported devices and confirm that users can complete the main Route Pay flow without workarounds.
